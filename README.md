Задание

Концепция задания:
	Нужно разработать 2 сервиса.
    1. Сервис представляет из себя файловое хранилище, позволяющее загружать/скачивать файлы и получать/изменять информацию о загруженных файлах.
    2. Сервис для операций над пользователем.
	Задание делится на три части, описанных в требованиях к функционалу. В первой части необходимо разработать основу сервисов с хранением данных в оперативной памяти, во второй добавить различный функционал, в третьей написать тесты и реализовать хранение данных в БД. 


Требования к архитектуре:
    • Необходимо разработать только API, пользовательский интерфейс не требуется
    • Сервисы должны быть написаны на Spring Boot
    • Сервисы должны использовать сборщик Gradle
    • Сервисы должны взаимодействовать асинхронно


Требования к функционалу:
Первая часть
    а. Модель данных хранилища должна иметь поля о дате загрузки файла, дате изменения, имени файла, типе файла, размере файла, статуса, идентификаторе пользователя, загрузившего файл, комментария и самого файла;
    б. Модель данных пользователя должна иметь поля о логине пользователя и его роли;
    в. Сервис-хранилище и сервис пользователей должны представлять из себя REST сервисы, обеспечивающие операции CRUD;
    г. Хранение файлов и их данных реализовать в оперативной памяти;
    д. Разработать метод возврата списка имен файлов;
    е. Все ответы от сервиса, как положительные, так и с ошибками, должны иметь формат JSON, за исключением скачивания файла.
Вторая часть 
    а. Разработать метод скачивания файла по прямой ссылке;
    б. Разработать метод, возвращающий список моделей данных (кроме самого файла, но с полем со ссылкой на скачивание файла);
    в. К предыдущему методу добавить возможность фильтрации
        ◦ по имени (возможно частичное вхождение)
        ◦ по дате изменения (диапазон от и до)
        ◦ по типу (без указания - все, при указании - список переданных типов)
    г. Добавить валидацию загружаемого файла по типу и размеру. Тип - распространенные форматы документов, размер - 15 Мб;
    д. Доработать метод загрузки файла: после загрузки файла, сервис должен асинхронно запустить проверку файла на вирусы (эмуляция). Пока файл проверяется, его статус помечается как "На проверке", и скачивание запрещено, пока статус не станет "Готов". Скан проходит 15 секунд;
    е. Доработать метод возврата списка имен (1Д): реализовав интеграцию между сервисами, сделать возможным возврат списка, передав на вход логин пользователя;
    ж. В сервисе пользователя реализовать метод, позволяющий получить ссылку на скачивание файлов по логину пользователя, загрузившего файл;
    з. Разработать метод скачивания нескольких указанных файлов в одном архиве. Загрузку файлов из места хранения реализовать параллельно;
    и. Сделать обработку различных Runtime Exceptions (ошибок обработки, валидации и т.д.)
    к. Обеспечить документирование API через Swagger
Третья часть
    а. Разработанные методы покрыть юнит-тестами (интеграционные будут плюсом)
    б. Пункт 1в (хранение файлов) реализовать в реляционной БД
    в. БД должна быть развернута в Docker
    г. Пункт 2в (поиск с фильтрацией) реализовать используя инструменты работы с БД
    д. Подключить инструмент миграции БД. Написать скрипты, осуществляющие миграцию разработанной схемы БД


Темы, изучаемые в процессе выполнения задания:
    1. Создание Gradle проекта и настройка минимального Spring Boot проекта
    2. Структура пакетов Spring Boot проекта
    3. Основные компоненты Spring Boot
    4. REST API, его основные принципы, способы реализации в Spring
    5. Асинхронная обработка, временная блокировка операций, многопоточное выполнение задач, thread-safe хранение статусов.
    6. Межсервисная интеграция
    7. Тестирование API с помощью Swagger и Postman
    8. Библиотека Jackson
    9. Обработка Runtime Exceptions в Spring Boot приложении
    10. Написание юнит-тестов
    11. Работа с реляционными БД
    12. Работа с Docker
    13. Составление запросов на выборку данных с фильтрацией
    14. Средства миграции БД
